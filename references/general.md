# 通用类笔记生成规则（自由模式）

## 适用范围
本参考规则适用于不限定学科领域的通用笔记生成场景。与编程类/非编程类规则不同，**本规则不预设任何章节模板或模块骨架**，由 AI 根据材料内容自行判断最佳结构和组织方式。

## 输入处理规则

1. **语音转文字稿处理**
   - 剔除口语助词、重复讲解、停顿语气词（uh/ok/ah 等），保留全部知识点、实操讲解、报错分析
   - 口头描述的逻辑、步骤、数据，补全为完整文字说明
   - 课堂口头强调的考试重点、注意点，在笔记中标出

2. **课件/现有笔记处理**
   - 复用原材料的章节编号和标题体系（如存在），统一全文格式风格
   - 课件中的图、表、例题逻辑，转化为文字说明或表格
   - 合并多份材料的同源知识点，去重但不删减有效信息

3. **视频文件处理**
   - 转写结果等同于语音稿，按上述规则处理

## 输出规则

### 语言模式适配
- **纯中文**：所有内容使用简体中文
- **中文 + 术语对照**：正文使用中文，关键术语附英文原文对照
- **纯英文**：全文使用标准专业英文表述

### Markdown 格式要求
- 使用标准 Markdown 语法，层级清晰（H1 → H2 → H3，最多不超过四级）
- 代码使用 ``` 围栏代码块并标注语言，行内代码使用反引号
- 重要关键词、概念、术语加粗标注
- 对比类信息优先使用表格呈现
- 枚举项使用无序/有序列表
- 公式可用行内 $ 或块级 $$ 按需选择

### 紧凑排版规则
- 同一标题下的任意两个内容元素（段落、代码块、表格、列表）之间最多保留**一个空行**
- 标题与其后第一个内容块之间不加空行（0 个空行）
- 列表项之间不插入空行，除非用于分隔语义上不同的组。**Markdown 表格的行与行之间绝对不能插入空行（空行会导致表格断裂，Obsidian 等渲染器只能识别第一行），表格前后各留一个空行即可。**
- 段落内部行间不插入空行，段落之间最多一个空行
- 不使用装饰性分隔线或多余的空行来"美化"排版

### 结构原则（自由裁量）
- **无预置模板**：根据材料内容自行判断最合适的章节划分、层级深度和组织逻辑
- **因材制宜**：理论为主则重定义与论证，应用为主则重案例与步骤，对比为主则重表格与辨析
- 每个知识点的展开方式由 AI 根据其在材料中的重要性和复杂度自行决定

## 不可妥协的底线
- **完整性**：100% 覆盖所有输入材料的知识点、例题、讲解、注意事项，严禁省略、简化核心内容
- **连贯性**：全文档逻辑连贯，章节、知识点衔接自然，风格统一
- **严谨性**：事实性内容、代码、公式严格遵循原文/原标准，无知识性错误
- **详略得当**：整体精炼但全面，语句简洁不啰嗦，细节无缺失
