
# 非编程类课程笔记生成规则（开卷考试速查模式）

## 目标
生成一份专为**开卷考试现场快速定位、理解和应用**的笔记。其核心原则是：
- **零遗漏**：覆盖所有材料中的知识点、注意点、例外情况。
- **极致索引**：提供多重检索入口（目录、关键词索引、公式表、代码片段索引）。
- **例题导向**：每个抽象概念必须配以可代入考试的例题/代码，并展示完整解题步骤。
- **精炼但完整**：去除废话，保留全部事实性信息，使用表格、列表、代码块压缩呈现。

---

## 笔记结构（强制）

### 1. 元数据头
# [课程名称] 开卷速查笔记
- 生成时间：YYYY-MM-DD
- 材料来源：语音稿/课件/现有笔记/其他（列出）
- 适用考试：开卷，允许携带打印/电子版
- 关键词索引：见文末附录
2. 全局目录（含二级标题）
按课程官方大纲或课件章节顺序组织。
每个章节标题后附该章节核心公式/代码片段页码（便于考试跳转）。
示例：


## 第3章 矩阵运算
  - 3.1 矩阵乘法 (p.12)
  - 3.2 逆矩阵 (p.14)
  - 3.3 应用：线性方程组 (p.16)
3. 章节内容模板（每章/每节自适应选用，不得遗漏任何内容）
以下为完整的六个子模板。根据每节知识点的性质灵活选用最合适的 3-5 个子模板，**材料中涉及该子模板相关内容的必须保留，不涉及的可跳过**。

判断指南：
- **定义/概念密集型**（如"正定矩阵定义"）→ 3.1 核心概念速览 + 3.2 关键公式 + 3.4 易错点
- **计算/应用型**（如"线性方程组求解"）→ 3.1 核心概念速览 + 3.2 关键公式 + 3.3 典型例题 + 3.4 易错点
- **对比辨析型**（如"梯度下降 vs 牛顿法"）→ 3.1 核心概念速览 + 3.5 快速记忆口诀/对比表
- **仅公式定理**（如"拉格朗日中值定理"）→ 3.1 核心概念速览 + 3.2 关键定理 + 3.3 典型例题
- **底线**：只要材料中有相关内容，对应的子模板就不能省略。
3.1 核心概念速览
用无序列表列出本节的 3~5 个核心关键词，每个词后紧跟一句话定义（加粗关键词）。
例：

正定矩阵：对称矩阵 A，满足任意非零向量 x，xᵀAx > 0。

Cholesky分解：正定矩阵 A = LLᵀ，L 为下三角。

3.2 关键公式/定理/代码块（按编号）
使用带编号的公式块或代码块，并附简短文字说明使用条件。
公式示例：

**公式 3.1** 矩阵乘法结合律  
(AB)C = A(BC)  
条件：A, B, C 可乘。注意：**不满足交换律**。
代码示例（Python/Java等）：

# 代码段 3.2：矩阵乘法实现（numpy）
import numpy as np
def mat_mul(A, B):
    return np.dot(A, B)   # 注意维度匹配
3.3 典型例题（带解题步骤）
至少 1 道例题，并明确展示思考路径。
结构：

题目（原文或简化）。

考点（对应公式/定理编号）。

标准解题步骤（分点列出，关键中间步骤加粗）。

最终答案（用框线或单独行突出）。
例：

**例题 3.1** 判断矩阵 A = [[2,1],[1,2]] 是否正定。
**考点**：公式 3.1（正定定义）
**步骤**：
1. 验证对称：A = Aᵀ ✓
2. 计算 xᵀAx = 2x₁² + 2x₁x₂ + 2x₂² = 2[(x₁+0.5x₂)² + 0.75x₂²] > 0
3. 因此正定。
**答案**：是正定矩阵。

⚠️ 求逆时需检查行列式是否为 0。

4.3 符号/缩写对照表（如适用）
学科专有符号或缩写（尤其英文课程）。

生成细则（针对 AI）
信息完整性：必须提取所有材料中的定义、定理、性质、算法步骤、注意事项、例题，不得省略。

语言精炼：避免重复叙述，用列表、表格替代长段落。

例题覆盖：每个小节至少包含 2 个典型例题（或 1 个复杂例题），并确保涵盖材料中提到的所有题型变体。

交叉引用：在正文中凡引用其他章节的公式/定理，须以超链接或括号注明编号（如 （详见公式 5.2））。

- **紧凑排版**：同一标题下的任意两个内容块（段落、代码围栏、表格、列表）之间最多保留一个空行。标题与其后第一个内容块之间不加空行。关键词索引（4.1）条目之间不加空行。公式/代码速查表（4.2）为单一紧凑表格。**Markdown 表格的行与行之间绝对不能插入空行（空行会导致表格断裂，Obsidian 等渲染器只能识别第一行），表格前后各留一个空行即可。**

排版优先级：

使用 ## 和 ### 分层，避免过深（不超过四级）。

重要关键词加粗，公式用 $$ 块或行内 $，代码用 ``` 围栏。

表格用于对比，列表用于枚举。
