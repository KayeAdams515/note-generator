# 编程类课程笔记生成规则（C/C++ 等）

## 一、适用范围
本参考规则专属 note-generator 技能下编程类课程笔记生成场景，输入材料包含课程语音转文字稿、课件、已有笔记、补充资料等；输出为标准 Markdown 格式结构化笔记，面向零基础 / 课堂复盘、期末复习、开卷查阅、代码练习场景，完整保留全部知识点、讲解细节、实操要点、语法规则、编程陷阱，不做内容删减。

## 二、整体格式与排版规范
1. **文档结构总规则**
   - 全局使用 Markdown 标准语法，层级清晰，严格按章节→小节→细分模块划分，对应课程原有章节、小节编号（如 9.8、9.9）。
   - 一级标题：`# 第X章 章节总标题`（对应课程大章节）；
   - 二级标题：`## X.X 小节名称`（对应课堂 Session / 小节号）；
   - 三级标题：固定使用下文规定的五大标准模块标题，不可修改、调换顺序；
   - 四级及以下标题按需使用，用于细分子知识点、例题、补充说明；
   - 专业关键字（class/const/static/friend/this 等关键字、运算符、访问修饰符）加粗标注；
   - 代码统一使用 ```cpp 代码块包裹，代码必须带详细注释，匹配课堂案例、例题；
   - 对比类知识点（如静态 / 非静态、const / 普通成员）优先使用 Markdown 表格呈现；
   - 如果有口语化内容（uh/ok/ah、重复语句、语气词），全部剔除，转换为标准书面编程表述。
   - **紧凑排版规则**：
     - 同一标题下的任意两个内容元素（段落、代码块、表格、列表）之间最多保留 **一个空行**。
     - 标题与其后第一个内容块之间不加空行（即标题行后直接跟内容，中间 0 个空行）。
     - 代码块紧接上文放置，最多间隔一个空行。
     - 列表项之间、表格行之间不插入空行，除非用于分隔语义上不同的组。
     - 五大模块之间（如"核心定义"段落结束到"### 2 语法示例"）最多间隔一个空行。

2. **自适应核心模块（按知识点性质灵活选用，不得遗漏任何内容）**
   每个独立语法 / 概念 / 知识点，从以下五大模块中选取 **2-4 个最合适的模块** 输出，保留模块间的原始顺序（不用的模块直接跳过，不输出空标题）。选择原则：**宁可多用不可漏信息，模块承载的知识点信息越多越要保留。**

   **模块选用指南：**
   - **纯概念/设计理念**（如"面向对象三大特性"、"什么是 RAII"）→ 核心定义 + 规则总结 + 易错点，可省略语法示例和代码演示
   - **纯语法规则**（如 const 修饰符位置、函数重载规则）→ 语法示例 + 代码演示 + 规则总结 + 易错点，核心定义可精简为一句话
   - **实战技巧/陷阱**（如"迭代器失效"、"悬空指针"）→ 代码演示 + 易错点 + 规则总结
   - **完整知识体系**（如"拷贝构造函数"、"虚函数机制"）→ 五模块全用
   - **判断底线**：只要材料中有该模块的相关内容，该模块就必须出现。仅当材料中确实不涉及该模块时方可省略。

   - **模块 1：核心定义**
     - 提取所有材料中对知识点的讲解、概念、作用、应用场景、设计目的；
     - 整合语音、课件、现有笔记中的全部解释，精炼语句，保证语义完整，无内容遗漏；
     - 区分基础概念、引申概念、课堂补充拓展内容。
     - 若知识点为纯语法/技巧型，可精简为一句话定义，不必扩写成段落。

   - **模块 2：语法示例**
     - 输出该知识点标准语法格式；
     - 拆分语法组成部分、语法限制、书写规范；
     - 区分合法写法、非法写法，标注语法报错原因；
     - 若存在多种重载、变体语法，全部逐一列出。
     - 若知识点无语法形态（如纯设计理念），可省略此模块。

   - **模块 3：代码演示**
     - 基于课堂案例、课件例题、语音描述的代码逻辑，编写完整可运行 C/C++ 代码；
     - 代码添加逐行关键注释，贴合课堂讲解逻辑；
     - 补充代码运行流程、预期输出结果、分步执行说明；
     - 存在多组示例（正常用法、边界用法、特殊用法）时，分块展示并分别说明。
     - 若材料中无任何代码相关内容，可省略此模块。

   - **模块 4：规则总结**
     - 提炼语法规则、使用约束、调用规则、访问规则、执行顺序（如构造 / 析构顺序、成员初始化顺序）；
     - 汇总课堂强调的编程规范、使用原则、最佳实践；
     - 涉及调用关系、权限关系、生命周期的规则，分点 / 表格梳理。

   - **模块 5：易错点**
     - 汇总材料中提及的编译错误、运行错误、逻辑错误、编程陷阱；
     - 每条错误标注：错误写法 + 问题原因 + 修正方案；
     - 包含语法坑、内存坑、封装破坏、权限误用、调用顺序错误等编程常见问题；
     - 课堂标注的考试易错点、改错题考点优先在此汇总。
     - 若材料中未提及任何易错信息，可省略此模块。

## 三、细分内容处理细则
1. **语音转文字稿处理**
   - 剔除口语助词、重复讲解、停顿语气词，保留全部专业知识点、实操讲解、报错分析；
   - 口头描述的代码、运行效果、调用顺序，补全为完整代码 + 运行结果；
   - 课堂口头强调「考试重点、大题 / 选择题 / 改错题」，在对应模块中标注并高亮；
   - 老师补充的拓展用法、特殊场景，全部整合进对应模块。

2. **课件 / 现有笔记处理**
   - 复用原有章节编号、标题体系，统一全文格式风格；
   - 课件中的例题、习题、示意图逻辑，转化为可运行代码 + 文字说明；
   - 合并多份材料的同源知识点，去重但不删减有效信息；
   - 课件中标注的考点、重难点，同步归类至「规则总结」「易错点」模块。

3. **语言模式适配**
   - 纯中文：所有内容使用简体中文，编程术语使用中文译名；
   - 中文 + 学科专有名词对照：正文使用中文，关键编程术语、关键字、语法搭配中文 + 英文原文对照标注；
   - 纯英文：全文使用标准计算机专业英文表述，语法、术语遵循 C/C++ 国际通用写法。

## 四、输出整体要求
- **完整性**：100% 覆盖所有输入材料的知识点、例题、讲解、注意事项，严禁省略、简化核心内容；
- **连贯性**：全文档逻辑连贯，章节、知识点衔接自然，风格统一；
- **实用性**：适配课堂复盘、知识点背诵、代码练习、期末复习、开卷查阅多场景；
- **严谨性**：代码语法、规则描述、错误分析严格遵循 C/C++ 标准，无知识性错误；
- **详略**：整体「精炼但全面」，语句简洁不啰嗦，细节无缺失。
